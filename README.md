# Моё портфолио

Адаптивный одностраничный сайт-портфолио, созданный на **HTML, CSS и JavaScript**.  
Проект демонстрирует семантическую верстку, карточки проектов, форму обратной связи, динамическую подгрузку контента и переключение темы с сохранением выбора в `localStorage` [web:35][web:33].

---

## О проекте

Этот сайт был создан как учебный проект и личное портфолио.  
В нём собраны основные разделы, которые обычно ожидаются от современного одностраничного сайта: шапка с навигацией, основной контент, карточки работ и контактная форма [web:35][web:33].

Проект удобно использовать как основу для:
- учебного задания по HTML/CSS/JS;
- личного портфолио;
- демонстрации навыков адаптивной верстки и работы с DOM;
- публикации на GitHub Pages [web:26][web:35].

- Семантическая структура страницы: `header`, `main`, `footer`.
- Шапка с навигацией по разделам.

## Основные разделы сайта

### Шапка с навигацией
В верхней части страницы находится логотип, навигационное меню и кнопка переключения темы.  
Ссылки ведут к основным блокам страницы: «Обо мне», «Проекты», «Контакты».

### Карточки проектов
Карточки отображаются в сетке и создаются в JavaScript из массива данных.  
Это позволяет легко добавлять новые проекты без ручного редактирования HTML.

### Кнопка «Показать ещё»
Кнопка подгружает дополнительные карточки, если они есть в массиве.  
Если проекты закончились, кнопка скрывается.

### Форма обратной связи
Форма открывается в модальном окне.  
При отправке:
- данные собираются в объект;
- email проверяется по регулярному выражению;
- результат выводится в `console.log`;
- окно закрывается после успешной отправки.

### Переключение темы
Кнопка в шапке меняет светлую и тёмную тему.  
Выбор сохраняется в `localStorage`, поэтому после обновления страницы тема остаётся прежней.

## Как запустить проект

### Локально
1. Скачай или клонируй репозиторий.
2. Открой файл `index.html` в браузере.
3. Проверь работу навигации, карточек, формы и кнопки темы.

### Через GitHub Pages
1. Загрузи файлы проекта в репозиторий GitHub.
2. Открой **Settings → Pages**.
3. В разделе **Build and deployment** выбери источник **Deploy from a branch**.
4. Укажи ветку `main` и папку `/root`.
5. Сохрани изменения.
6. Подожди, пока GitHub создаст публичную ссылку [web:21][web:26][web:17].

## Как работает логика сайта

### Динамические карточки
В JS есть массив проектов.  
Функция рендеринга берет элементы из массива и вставляет их в блок карточек через `innerHTML`.

### Тема
Тема меняется добавлением или удалением класса `dark-mode` у `body`.  
Значение темы записывается в `localStorage`, чтобы не сбрасываться после перезагрузки.

### Форма
При отправке формы используется `FormData`.  
Потом данные превращаются в объект и проходят проверку email.  
Если email некорректный, показывается сообщение об ошибке.
